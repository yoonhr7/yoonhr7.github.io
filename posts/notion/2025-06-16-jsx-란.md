<aside data-icon="💡">

JSX는 JavaScript XML의 줄임말로, React에서 UI를 정의할 때 사용하는 문법이다. JavaScript 안에서 HTML처럼 생긴 코드를 쓸 수 있게 해주는 문법 확장(Syntax Extension)이라고 보면 된다.

</aside>


### 핵심 특징

- **HTML처럼 생겼지만 실제로는 JavaScript 문법**이다.
- JSX 코드는 Babel 등의 트랜스파일러에 의해 `React.createElement()` 같은 함수 호출로 변환된다.
- JSX 안에서는 `{}` 중괄호를 사용해 JavaScript 표현식을 쓸 수 있다

```javascript
const name = "Yoon";
const element = <h1>Hello, {name}</h1>;
```


---


### 주의할 점

- HTML과 아주 비슷하지만 **class 대신** **`className`**, **for 대신 `htmlFor`*처럼 이름이 조금 다른 속성들도 있다.
- JSX를 쓸 땐 꼭 **하나의 부모 요소로 감싸야** 한다.

```javascript
// 이렇게 하면 에러!
return (
  <h1>Title</h1>
  <p>Paragraph</p>
);

// 이렇게 해야 함
return (
  <div>
    <h1>Title</h1>
    <p>Paragraph</p>
  </div>
);
```

### 🆚 Vue 와 비교


| 비교 항목    | React                     | Vue (기본)                    | Vue (JSX 사용 시)   |
| -------- | ------------------------- | --------------------------- | ---------------- |
| 기본 문법    | JSX                       | 템플릿 문법                      | JSX 가능 (설정 필요)   |
| 중괄호로 JS? | `{}` 안에 JS 표현식 사용         | `{{ }}` 안에 값 출력             | `{}` 사용          |
| 조건/반복    | JS 문법 (`map`, `if`, `&&`) | `v-if`, `v-for`, `v-show` 등 | JS 문법            |
| 대표 장점    | JavaScript 중심             | HTML에 가깝고 직관적               | 고급 사용자에게 유연한 선택지 |
