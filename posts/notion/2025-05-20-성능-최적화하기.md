## 1. 이미지 최적화


### 1) loading 속성 이용하기

- HTML `<img>` 또는 `<iframe>` 태그에서 사용하는 속성으로, **브라우저가 해당 리소스를 언제 로드할지** 결정하는 방법을 지정합니다.

    | 속성                | 의미                | 사용 용도                    |
    | ----------------- | ----------------- | ------------------------ |
    | `loading="eager"` | 즉시 로드             | 처음에 바로 보여야 할 이미지         |
    | `loading="lazy"`  | 나중에 (스크롤 근처에서) 로드 | 아래쪽 콘텐츠, 썸네일 등 지연 가능 이미지 |


### 2) fetchpriority

- HTML 요소(주로 `<img>`, `<link>`, `<script>` 등)에 사용하는 **리소스 로딩 우선순위 힌트**입니다.
- `fetchpriority`는 브라우저가 리소스를 다운로드할 때 어떤 걸 **먼저** 가져올지 결정하는 데 도움을 줍니다.
    - `high` – **가장 높은 우선순위**로 로딩
    - `low` – **가장 낮은 우선순위** (백그라운드로 천천히)
    - `auto` – 브라우저가 알아서 판단 (기본값)

    | 상황                                   | 써야 할지 여부       |
    | ------------------------------------ | -------------- |
    | 화면 맨 위 중요한 이미지 (ex. Hero 배너, 브랜드 로고) | ✅ `high` 권장    |
    | 화면 아래 썸네일, 스크롤해야 보이는 콘텐츠             | ❌ `low` 또는 기본값 |
    | 광고, 통계 스크립트 등 후순위 리소스                | ✅ `low` 권장     |


`loading="eager"` vs `fetchpriority="high"`

- `loading`은 **"언제 로드할지"**
- `fetchpriority`는 **"얼마나 빨리 로드할지"**

<aside data-icon="👉">

- 대부분의 **최신 브라우저 (특히 Chrome)** 는 **`loading="lazy"`** **우선**으로 처리합니다.
- 즉, **lazy가 우선되므로 fetchpriority는 무시**되거나 **효과가 거의 없음**.

</aside>


| 조합                                         | 의미                   | 효과                       |
| ------------------------------------------ | -------------------- | ------------------------ |
| `loading="eager"` + `fetchpriority="high"` | 지금 보여야 하니까 즉시 불러와라!  | ✅ 잘 작동                   |
| `loading="lazy"` + `fetchpriority="low"`   | 느긋하게 불러와도 된다         | ✅ 잘 작동                   |
| `loading="lazy"` + `fetchpriority="high"`  | 나중에 보여줄 건데 지금 미리 받아? | ❌ 거의 효과 없음 (브라우저가 무시 가능) |
