### 1. vue-i18n 설치


```shell
npm install vue-i18n@9
```


### 2. 언어 파일 폴더 생성


```plain text
src/
├─...
├─ locales/
│  ├─ en.json
│  ├─ ko.json
├─...
```


### 3. 언어 파일 생성


 `src/locales/ko.json`


```json
{
  "navigation": {
    "home": "홈",
    "company": "회사소개",
    "business": "사업영역"
  },
  "common": {
    "loading": "로딩 중...",
    "error": "오류가 발생했습니다",
    "success": "성공적으로 완료되었습니다"
  }
}
```


`src/locales/en.json`


```json
{
  "navigation": {
    "home": "Home",
    "company": "Company",
    "business": "Business"
  },
  "common": {
    "loading": "Loading...",
    "error": "An error occurred",
    "success": "Successfully completed"
  }
}
```


### 4. i18n 설정 파일 생성 


`src/i18n/index.js`


```javascript
import { createI18n } from 'vue-i18n'
import ko from '../locales/ko.json'
import en from '../locales/en.json'

const messages = {
  ko,
  en
}

const i18n = createI18n({
  locale: 'ko',
  fallbackLocale: 'ko',
  messages,
  legacy: false,
  globalInjection: true
})

export default i18n
```


### 5. main.js에 i18n 연결


 `src/main.js`


```javascript
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'
import { createPinia } from 'pinia'
import i18n from './i18n'  // 추가

const app = createApp(App)

app.use(createPinia())
app.use(router)
app.use(i18n)  // 추가

app.mount('#app')
```


### 6. 언어 전환 컴포저블 생성 (선택사항) 


`src/composables/useLanguage.js`


```javascript
import { ref, computed } from 'vue'
import { useI18n } from 'vue-i18n'

const currentLanguage = ref('ko')

export function useLanguage() {
  const { locale, t } = useI18n()

  const availableLanguages = [
    { code: 'ko', name: '한국어' },
    { code: 'en', name: 'English' }
  ]

  const changeLanguage = (langCode) => {
    if (availableLanguages.some(lang => lang.code === langCode)) {
      locale.value = langCode
      currentLanguage.value = langCode
      localStorage.setItem('preferred-language', langCode)
    }
  }

  return {
    currentLanguage,
    availableLanguages,
    changeLanguage,
    t
  }
}
```


### 7. 컴포넌트에서 사용 방법 


```html
<template>
  <div>
    <h1>{{ $t('navigation.home') }}</h1>
    <p>{{ $t('common.loading') }}</p>
  </div>
</template>

<script setup>
import { useI18n } from 'vue-i18n'
const { t } = useI18n()

// 또는 컴포저블 사용
import { useLanguage } from '@/composables/useLanguage'
const { t, changeLanguage } = useLanguage()
</script>
```


### 8. 언어 전환 컴포넌트 (선택사항)


```javascript
<template>
  <select :value="currentLanguage" @change="handleLanguageChange">
    <option v-for="lang in availableLanguages" :key="lang.code" :value="lang.code">
      {{ lang.name }}
    </option>
  </select>
</template>

<script setup>
import { useLanguage } from '@/composables/useLanguage'

const { currentLanguage, availableLanguages, changeLanguage } = useLanguage()

const handleLanguageChange = (event) => {
  changeLanguage(event.target.value)
}
</script>
```


### 참고) 컴포저블 사용유무의 차  


| 구분         | **useI18n**                | **useLanguage (composable)**                       |
| ---------- | -------------------------- | -------------------------------------------------- |
| **정의**     | vue-i18n에서 제공하는 기본 훅       | useI18n을 감싸서 프로젝트 전용 로직을 추가한 함수                    |
| **제공 기능**  | 번역(`t`), locale 접근 등 기본 기능 | 번역 + 언어 전환 함수, localStorage 저장, 브라우저 언어 감지 등 확장 기능 |
| **장점**     | 단순, 가볍고 바로 사용 가능           | 재사용성 높음, 중복 로직 제거, 프로젝트 규칙 일관성 유지                  |
| **단점**     | 반복 로직 발생 가능, 커스터마이징 어려움    | 초기 구현 필요, 관리 대상 늘어남                                |
| **적합한 경우** | 번역만 빠르게 쓰고 싶을 때            | 언어 변경/저장/규칙 관리 등 프로젝트 단위 제어가 필요할 때                 |
