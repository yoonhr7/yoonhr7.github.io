### 4. 환경변수 관리


Nuxt 3은 **Vite 기반 + Nitro 서버 런타임**을 사용하기 때문에

- **빌드 타임 변수 (****`import.meta.env`****)** → `VITE_` 접두사 필요
- **런타임 변수 (****`useRuntimeConfig`****)** → `NUXT_` 접두사 필요

그래서 Vercel에서 환경변수를 인식시키고 싶다면 **Nuxt가 허용한 접두사**를 써야 합니다.


❎ **잘못된 예**


Vercel에 이렇게 등록하면 Nuxt 런타임에서 **절대 못 읽음**


| Key            | Value        |
| -------------- | ------------ |
| `NOTION_TOKEN` | `secret_xxx` |

> Nuxt는 NOTION_TOKEN이 존재하더라도 runtimeConfig에 주입하지 않음

✅ 올바른 예


```javascript
export default defineNuxtConfig({
  runtimeConfig: {
    notionToken: process.env.NUXT_NOTION_TOKEN,
    resendApiKey: process.env.NUXT_RESEND_API_KEY,
    public: {}
  }
});
```
