## 1. 공통으로 SEO 설정하기


```typescript
// nuxt.config.ts
export default defineNuxtConfig({
    app: {
        head: {
            title: "웹사이트 이름",
            meta: [
		            { charset: "utf-8" }, 
		            { name: "viewport", content: "width=device-width, initial-scale=1.0" }, 
		            { name: "description", content: "웹사이트 설명입니다" }, 
		            { property: "og:type", content: "website" }, 
		            { property: "og:site_name", content: "주소" }, 
		            { property: "og:title", content: "웹사이트 이름" }, 
		            { property: "og:image", content: "/og-image.jpg" }, 
		            { name: "twitter:image", content: "/og-image.jpg" }, 
		            { property: "og:url", content: "https://www.thingshub.kr" }, 
		            { property: "og:description", content: "웹사이트 설명입니다" }, 
		            { name: "naver-site-verification", content: "코드" }
		        ],
            link: [
                { rel: "icon", type: "image/x-icon", href: "/favicon.ico" }, // Nuxt에서는 /public/favicon.ico
            ],
            script: [
                {
                    type: "application/ld+json",
                    innerHTML: `
                    {
                        "@context": "https://schema.org",

                        "url": "https://www.example.com",

                    }
                    `,
                },
            ],
        },
    },
  });
```


### 1) script(ld+json) 작성하기


```typescript
script: [
  {
    type: 'application/ld+json',
    children: JSON.stringify({
      "@context": "https://schema.org",
      "@type": "Organization",
      ...
    }),
    tagPosition: 'head', // 선택적: 명확하게 head에 넣겠다는 의미
  }
]
```

- 일반적인 방식 (`innerHTML`)

    ```typescript
    script: [
      {
        type: "application/ld+json",
        
    innerHTML
    : `...`, // 직접 문자열을 넣는 방식
      }
    ]
    ```


    이 방식은 Nuxt 3에서는 **Vue의 DOM sanitizer**에 걸릴 수 있기 때문에 동작은 하지만 **경고 메시지가 뜨거나**, `innerHTML`이 제대로 렌더링되지 않는 경우도 있습니다.

- 안전한 방식 (`useHead + children`)

    ```typescript
    useHead({
      script: [
        {
          type: 'application/ld+json',
          
    children: JSON.stringify({
    
            "@context": "https://schema.org",
            "@type": "Organization",
            ...
          
    }),
    
          
    tagPosition: 'head',
     // 선택적: 명확하게 head에 넣겠다는 의미
        }
      ]
    })
    ```


    이 방식은 Nuxt에서 **정식으로 권장하는 형태**이며, 다음과 같은 장점이 있습니다:

    - Vue/Nuxt에서 내부적으로 **XSS 방지** 처리를 피해 안전하게 삽입됨
    - **Hydration 이슈 없음** (SSR/CSR 모두 호환)
    - `children` 속성은 Vue가 `innerHTML` 대신 사용하는 안전한 텍스트 삽입 방식
    > 요약: innerHTML은 긴급하게 쓸 수는 있지만, Nuxt 3에서는 useHead + children 조합이 공식적이고 안전한 방법입니다.

### 2) meta 와 script(ld+json)의 차이


| 항목           | `meta`         | `script` (ld+json)  |
| ------------ | -------------- | ------------------- |
| 포맷           | HTML 태그        | JSON 데이터            |
| 사용 목적        | SEO, 소셜미디어, 인증 | 검색엔진 이해를 돕는 구조화 데이터 |
| 사람이 봐도 이해 가능 | 대부분 가능         | JSON 구조로만 구성됨       |
| 검색 결과에 영향    | 있음 (기본적인 내용)   | 있음 (리치 결과에 더 큰 영향)  |


## 2. 페이지 별로 SEO 설정하기


```plain text
// 메인 페이지 SEO 메타 정보
useHead({
    title: "브라우저에 나오는 제목",
    meta: [{ name: "description", content: "페이지 설명" }],
    script: [
        {
            type: "application/ld+json",
            children: JSON.stringify({
                "@context": "https://schema.org",
                "@type": "WebPage",
                name: "페이지 이름",
                url: "https://www.example.com/",
                description: "페이지 설명",
            }),
        },
    ],
});
```

## 4. robot.txt 작성하기


```plain text
# robot.txt

# 모든 검색엔진 크롤러에 대한 설정
User-agent: *

# 사이트 전체를 허용(Allow)하되, 특정 경로만 차단(Disallow)하는 방식
Allow: /

# 예시: 관리자 페이지나 불필요한 경로 등을 차단
Disallow: /admin/

# 사이트맵 위치 표기
Sitemap: https://www.example.com/sitemap.xml
```


### 1) 기본구조

- `User-agent`: 어떤 크롤러에게 적용할지 지정 (*는 모든 봇)
- `Disallow`: 접근을 막고 싶은 경로
- `Allow`: 접근을 허용하고 싶은 경로
- `Sitemap` : 사이트맵 위치
